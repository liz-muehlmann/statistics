2.1 What is Statistical Learning?

  • XXX denotes the input variable (aka: predictor or independent variable)
  • YYY denotes the output variable (aka: response or dependent variable)
  • The relationship between X and Y can be written as:
    Y=f(X)+ϵY=f(X)+\epsilonY=f(X)+ϵ
  • fff is a fixed, but unknown function of XXX and ϵ\epsilonϵ is the error term.
  • ϵ\epsilonϵ is independent of XXX and has a mean of 0.
    The function fff may take more than one input variable. (e.g. income ( yyy ) as function of education ( X1X_1X1​ ) and seniority (X2X_2X2​)).

Statistics is all about ways to estimate fff

2.1.1 Why Estimate fff?

  1. Prediction: this is when we know the values of XXX, but can't easily determine YYY.
  • y^=f^(x)\hat{y} = \hat{f}(x)y^​=f^​(x)
  • Y^\hat{Y}Y^ is the resulting predicting for YYY (aka the predicted response)
  • f^\hat{f}f^​ is the estimate for fff.
    • It is a black box - where we don't really care about f^\hat{f}f^​ as long as it gives accurate predictions for YYY.

Generally, f^\hat{f}f^​ will not be a perfect estimate fff, as a result the inaccuracy will introduce some error.

Types of Error:

  • reducible error: can be reduced to improve the accuracy of f^\hat{f}f^​ by using better statistical methods.
  • irreducible error: since YYY is a function of ϵ\epsilonϵ, not all of the error can be reduced. Therefore there will always be ϵ\epsilonϵ

Reasons that ϵ\epsilonϵ is not zero

  • ϵ\epsilonϵ is not zero because it might include variables that are useful in predicting YYY.
    • Since we don't measure these unincluded variables they can't be predicted using fff.
  • ϵ\epsilonϵ may also contain unmeasurable variation, which also can't be predicted using fff

So if we have estimate f^\hat{f}f^​ and predictors XXX we get the prediction: Y^\hat{Y}Y^ = f^(X)\hat{f}(X)f^​(X).

If we assume that f^\hat{f}f^​ and XXX are fixed:

E(Y−Y^)2=E[f(X)−ϵ−f^(X)]2=[f(X)−f^(X)]2+var(ϵ)\begin{equation} \begin{split} E(Y-\hat{Y})^2 & = E[f(X) - \epsilon - \hat{f}(X)]^2\cr & =[f(X)-\hat{f}(X)]^2 + var(\epsilon) \end{split} \end{equation}E(Y−Y^)2​=E[f(X)−ϵ−f^​(X)]2=[f(X)−f^​(X)]2+var(ϵ)​​​

  • E=(Y−Y2)E=(Y-Y^2)E=(Y−Y2) is the expected value of the squared difference between the predicted value and actual value of YYY.
  • Var(ϵ)Var(\epsilon)Var(ϵ) is the variance associated with the error term ϵ\epsilonϵ

The irreducible error gives an upper bound on the accuracy of our prediction for YYY & will almost always be unknown in practice.

  1. Inference
  • Inference is when we want to know how YYY is affected by change in the predictors, X1...XpX_1...X_pX1​...Xp​, but aren't necessarily interested in making predictions for YYY.
  • the goal is to understand the relationship between XXX and YYY. How YYY changes as a function of X1...XpX_1...X_pX1​...Xp​
  • f^\hat{f}f^​ can't be treated as a black box because we have to know its exact form.
  • linear models are useful for inference.

Inference is useful for:

  • determining which predictors are associated with the outcome.
  • determining the relationship between the outcome and each predictor.
  • determining whether the relationship between YYY and each predictor can be summarized using a linear equation or whether the relationship between the two is more complicated.

2.1.2 How do we Estimate fff?

  • nnn is the number of data points or observations we have.
  • training data is a subset of the data we have that we use to train (or teach) the method how to estimate fff.
  • We apply a statistical learning method to the training data in order to estimate fff.
  • xijx_{ij}xij​ is the value of the jjjth predictor for observation iii.
    iii = 1, 2,..., nnn and jjj = 1, 2,...,ppp
    yiy_iyi​ is the response variable for iiith observation.
  • The training data would be (xi,yi),(x2,y2),...(xn,yn){(x_i,y_i), (x_2, y_2),...(x_n,y_n)}(xi​,yi​),(x2​,y2​),...(xn​,yn​), where x=(xi1,xi2,...,xip)Tx = (x_{i1}, x_{i2},..., x_{ip})^Tx=(xi1​,xi2​,...,xip​)T.

The goal is to find f^\hat{f}f^​ such that Y≈f^(X)Y \approx \hat{f}(X)Y≈f^​(X) for any observation (X,Y)(X,Y)(X,Y)

There are two statistical learning methods we can use:

  1. Parametric: to estimate fff we only need to estimate one set of parameters.
    • the problem is that it will usually not match the true unknown form of fff
    • if the model is too far off from the true fff (or the fff using all the observations), the estimate will be poor
    • to solve poor fit, we can use more flexible models. But more flexible models requires estimating more parameters.
    • more complex models can lead to overfitting: which means they follow the errors too closely.
    • these involve a two-step model-based approach.

Step 1 We make an assumption of fff's form.
For example, if fff is linear:

  • f(X)=β0+β1X1+β2X2+...+βpXpf(X) = \beta_0 + \beta_1X_1 + \beta_2X_2 + ... + \beta_pX_pf(X)=β0​+β1​X1​+β2​X2​+...+βp​Xp​
  • If fff is linear, you only need to estimate the coefficients β0+β1X1+β2X2+βpXp\beta_0 + \beta_1X_1 + \beta_2X_2 + \beta_pX_pβ0​+β1​X1​+β2​X2​+βp​Xp​

Step 2 We use the training data to fit or train the model.
For the linear model we want to estimate: Y≈β0+β1X1+β2X2+...+βpXpY \approx \beta_0 + \beta_1X_1 + \beta_2X_2 + ... + \beta_pX_pY≈β0​+β1​X1​+β2​X2​+...+βp​Xp​

One method of fitting the model is (ordinary) least squares

  1. Non-parametric:

    • do not make explicit assumptions about the functional form of fff.
    • these methods try to get as close to the data points without being too rough.
    • since they don't assume particular form of fff, they can fit a wider range of shapes for fff.
    • will fit the data better since it does not assume the form of fff.
    • requires substantially more observations in order to get an accurate estimate for fff than parametric approaches.

2.1.3 The Trade-off Between Prediction Accuracy and Model Interpretability

Some methods are less flexible because they can produce only a small range of shapes to estimate fff (e.g. Linear regression can only create linear functions.)

  • less flexible models are better for inference because they are more interpretable.
  • it's easier to understand the relationship between YYY and X1,X2,...XpX_1, X_2,...X_pX1​,X2​,...Xp​
    More flexible models include thin plate splines can generate a wider range of possible shapes to estimate fff.

2.1.4 Supervised vs. Unsupervised Learning

Supervised Learning for each observation of the predictor measurements xi,i=1,...,nx_i, i=1,...,nxi​,i=1,...,n there is an associated response to the measurement yiy_iyi​.

  • Includes: logistic regression, GAM, Boosting, and support vector machines

Unsupervised Learning is more complicated because for every observation i=1,...,ni=1,...,ni=1,...,n there's a vector of measurements xix_ixi​, but no associated response yiy_iyi​.

  • it is called unsupervised because we have no response variable yyy to supervise our analysis.
    • Includes: cluster analysis (do the observations fall into relatively distinct groups?)

Semi-supervised Learning is when you have observations for the predictors, but the corresponding measurements for the responses are less available.

2.1.5 Regression vs. Classification Problems

Quantitative Variables are number variables.

  • problems involving Quantitative data are called regression problems.

Qualitative Variables (aka categorical) take on one of a number of classes or categories.

  • problems involving Qualitative data are called classification problems.

The difference between the two is not that clear cut and some methods (K-nearest-neighbors and boosting) can be used for both quantitative adn qualitative data.

Most statistical methods are based on whether the response is qualitative or quantitative.

2.2 Assessing Model Accuracy

There is no best method in statistical learning. The data you have determines what kind of method you can (and should!) use.

2.2.1 Measuring the Quality of Fit

Measuring the quality of fit is a quantification of the extent to which the predicted response value for a given observation is close to the true response value of the observation.

For regressions, the most common measure is the mean squared error (MSE).

  • MSE=1n∑i=12(yi−f^(x1)2)MSE = \frac {1}{n} \displaystyle \sum_{i=1}^{2}(y_i - \hat{f}(x_1)^2)MSE=n1​i=1∑2​(yi​−f^​(x1​)2)
    • f^(xi)\hat{f}(x_i)f^​(xi​) is the prediction f^\hat{f}f^​ gives for the iiith observation.
  • a small test MSEMSEMSE is the goal
    • A small MSEMSEMSE means that the predicted responses are close to the true responses.
  • We're interested in how well our method works with previously unseen data (i.e., not the training data.)
  • If we have training observations, which we use to fit obtain an estimate of f^\hat{f}f^​
    • (x1,y1),(x2,y2),...,(xn,yn){(x_1, y_1), (x_2, y_2), ..., (x_n, y_n)}(x1​,y1​),(x2​,y2​),...,(xn​,yn​)
  • We then compute f^(x1),f^(x2),...,f^(xn)\hat{f}(x_1), \hat{f}(x_2), ..., \hat{f}(x_n)f^​(x1​),f^​(x2​),...,f^​(xn​)
    • If these are approximately equal to y1,y2,...,yny_1, y_2, ..., y_ny1​,y2​,...,yn​ then the MSEMSEMSE is small.
  • What we want to know is if f^(x0)≈y0\hat{f}(x_0) \approx y_0f^​(x0​)≈y0​ where x0x_0x0​ and y0y_0y0​ is a previously unseen test observation not used to train the statistical learning method. Essentially, a new data point.
    • This is the test MSEMSEMSE which is what we're interested in and the lower the better.
  • For a large number of observations we can compute:
    • Ave(y0−f^(x0))2Ave(y_0 - \hat{f}(x_0))^2Ave(y0​−f^​(x0​))2
    • The average squared prediction error for the test observations: (x0,y0)(x_0, y_0)(x0​,y0​)
    • The smaller the number the better
  • DO NOT rely on a method that results in a small training MSEMSEMSE even though the test and training MSEMSEMSE are closely related because most methods try to make a small training MSEMSEMSE which does not always result in a small test MSEMSEMSE.
    • Overfitting Is a method that gives as small training MSEMSEMSE but a large test MSEMSEMSE.
      • It means that the method may be picking up patterns that are just random chance
  • The training MSEMSEMSE will generally be smaller than the test MSEMSEMSE.
  • Cross-Validation is a method for estimating the test MSEMSEMSE using the training data.

2.2.2 The Bias Variance Trade-Off

The expected test MSEMSEMSE (for a given value, x0x_0x0​) is the sum of three fundamental quantities:

  1. The variance of f^(x0)\hat{f}(x_0)f^​(x0​)
  2. The squared bias of f^(x0)\hat{f}(x_0)f^​(x0​)
  3. The variance of the error terms ϵ\epsilonϵ.
  • this is:
    • E(y0−f^(x0))2=Var(f^(x0))+[Bias(f^(x))]2+Var(ϵ)E(y_0 - \hat{f}(x_0))^2 = Var(\hat{f}(x_0)) + [Bias(\hat{f}(x_))]^2 + Var(\epsilon)E(y0​−f^​(x0​))2=Var(f^​(x0​))+[Bias(f^​(x)​)]2+Var(ϵ)
      • Where E(y0−f^(x0))E(y_0 -\hat{f}(x_0))E(y0​−f^​(x0​)) is the expected test MSEMSEMSE.
        • It is the average test MSEMSEMSE "we'd obtain if we repeatedly estimated fff using a large number of training sets and tested each at x0x_0x0​"
  • The test MSEMSEMSE will never be below Var(ϵ)Var(\epsilon)Var(ϵ) (the irreducible error)
  • Our goal is to have low variance and low bias through the bias-variance tradeoff
    • Variance: the amount that f^\hat{f}f^​ would change if we estimated it using a different training data set.
      • Ideally, we would not like f^\hat{f}f^​ to vary much between data sets.
      • a high variance would mean that small changes in the training data results in large changes in f^\hat{f}f^​.
      • More flexible methods will have higher variance.
    • Bias: is the error that is introduced by approximating real life using a simple model.
      • The closer real life is to the model used, the less bias there will be.
      • More flexible methods tend to have less bias.

The Bayes Classifier

The test error rate is minimized (on average) by a simple classifier that assigns each observation to the most likely class, given its predictor values.

  • we should assign a test observation with predictor vector x0x_0x0​ to the class jjj for which the below is the largest
    • Pr(Y=j∣X=x0)Pr(Y = j | X = x_0)Pr(Y=j∣X=x0​)
      • This is a conditional probability equation.
      • It is the probability that Y=jY = jY=j, given the observed predictor vector x0x_0x0​

This is the Bayes classifier. In a two-class problem it says that there are only two possible response values.

  • it corresponds to predicting the first class if Pr(Y=1∣X=x0)>0.5Pr(Y = 1 | X = x_0) > 0.5Pr(Y=1∣X=x0​)>0.5, and class two otherwise.
  • The Bayes decision boundary is where the points have a probability of exactly 50%.
    • The Bayes classifier is determined by this boundary.
  • The Bayes classifier produces the Bayes error rate which is the lowest possible test error rate.
    • The overall Bayes Error Rate: 1−E(majxPr(Y=j∣X))1 - E( m\underset{j}axPr(Y = j | X))1−E(mja​xPr(Y=j∣X))
      • where the expectation averages the probability over all possible values of XXX
      • It is analogous to the irreducible error.

Since we will never know the conditional distribution of YYY given XXX in a real data set, the Bayes classifier is an unattainable gold standard against which we compare other models.

K-Nearest Neighbors

Attempts to estimate the conditional distribution of YYY given XXX, then classifies a given observation to the class with the highest estimated probability.

  • Given a positive integer KKK and a test observation x0x_0x0​, KNNKNNKNN:

    1. Applies Bayes rule and classifies the test observation x0x_0x0​ to the class with the largest possibility.
  • The user picks the value for KKK.
    • A low KKK will produce a flexible model, but may have low bias but very high variance.
    • The higher the KKK, the less flexible the model, which will result in low variance but high bias.
  • The training error and the test error do not have a strong relationship.
